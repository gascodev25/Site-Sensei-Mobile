# **Product Requirements Document (PRD)**  
## **Warehouse & Service Management System**  
**Version:** 2.0 (Rewritten for Clarity & Execution)  
**Author:** Lewis  
**Date:** 2025-04-05

> **Purpose**: This document defines the scope, functionality, and architecture for a field service and warehouse control system. It separates **V1 (Build Now)** from **V2/V3 (Architect For)** and removes ambiguity from mixed notes and inspirations.

---

## 1. Overview

### Problem
Field service operations rely on spreadsheets and manual tracking. Equipment in the field, consumables usage, and service scheduling lack real-time visibility. Invoicing is delayed. Stockouts occur. Contract renewals are missed.

### Vision
A unified control center for:
- Scheduling contract installations and recurring services  
- Tracking equipment and consumables from warehouse to field  
- Managing team assignments and stock issuance  
- Flagging low stock, expiring contracts, and missed services  
- Preparing for mobile, routing, and accounting integration  

This is not a calendar. It’s an **operations nerve center**.

## 2. Goals & Success Metrics (V1)

| Goal | Success Metric |
|------|----------------|
| Reduce missed services | <5% of scheduled services marked "missed" monthly |
| Eliminate stockouts | Low stock alerts triggered 7+ days before depletion |
| Speed up invoicing | 90% of completed services marked for invoicing within 24h |
| Reduce admin time | 40% reduction in time spent scheduling and tracking manually |
| Improve contract retention | 80% of expiring contracts reviewed 30 days in advance |

## 3. User Roles & Personas

### Primary Users

| User | Role | Pain Points |
|------|------|-----------|
| **Ops Manager (Sarah)** | Runs daily ops, assigns teams, approves services | Manual scheduling, no real-time stock visibility |
| **Warehouse Clerk (Jamal)** | Manages stock in/out, tracks field equipment | No sync with field teams, can’t see what’s issued |

| Role | Permissions | Can Create Users? |
|------|------------|-------------------|
| `super_user` | Full access | ✅ Yes |
| `general_manager` | Full access | ✅ Yes |

| `admin` | Edit clients, equipment, CSV upload | ❌ No |
| `warehouse_clerk` | View/adjust stock, return items | ❌ No |

> **Note**: Users can have multiple roles (e.g., `ops_manager,admin`).

## 4. V1 Scope: Core Functionality

### 4.1 Client Management
- Add/edit/delete clients (name, address, contact)  
- Bulk upload via CSV with **preview and validation**  
- Address entry:
  - OSM autocomplete with **re-geocoding**  
  - Manual fallback with **map click to set lat/lng**  
  - Store: `address_text`, `latitude`, `longitude`, `city`, `postcode`, `country`  
- Searchable by address (fuzzy match on `address_text`)

### 4.2 Equipment & Consumables Management
- Add/edit/delete equipment (name, stock code, price, install date)  
- Add/edit/delete consumables (name, stock code, price, min stock level)  
- Link multiple consumables to each equipment  
- Bulk upload via CSV with **preview and validation**  
- Reserve fields: `barcode`, `qr_code` (for V2 scanning)  

### 4.3 Team Management
- Add/edit/delete team members (name, phone, skill)  
- Create service teams by assigning members  
- Bulk upload members via CSV with **preview and validation**  

### 4.4 Service Management
- Create **Contract Installation**:
  - Assign client, equipment, consumables (linked + standalone)  
  - Dropdowns with text search  
  - Assign team, set install date  
- Convert to **Service Contract**:
  - Retains all data  
  - Set service intervals: 7, 14, 30, 60, 90, 180 days, or Once-Off  
  - Set contract length (months)  
  - Recurring services only Mon–Fri (Sat/Sun require manual override)  
- **Visual indicator (🔁)** on recurring services in calendar and lists  
- Calendar view:
  - Drag-and-drop rescheduling  
  - Prompt: “Change this instance or all future services?” (only for recurring)  
  - Color-coded: amber (scheduled), green (completed), red (missed)  
- Mark service as completed:
  - **Prompt to return unused stock** (with “Skip” option)  
  - Set `completed_at`, log action  
- Reporting: filter by interval, team, completion status

### 4.5 Warehouse Stock Control
- Track equipment: in warehouse, in field, issued to teams  
- Track consumables: current stock, min level, forecasted usage (daily/weekly/monthly/quarterly)  
- Deduct stock when service is scheduled and assigned to team  
- Return unused stock manually (not required to complete service)  
- Reporting: export equipment in field, warehouse stock, forecasted consumables  

### 4.6 Invoicing (V1: Manual)
- Mark service as “ready to invoice” on completion  
- Dashboard shows “Services Ready to Invoice”  
- User manually creates invoice in Sage One  
- Returns to app, marks as **invoiced** with:
  - `invoiced_status = 'invoiced'`  
  - `invoiced_by = user_id`  
  - `last_invoice_sync = NOW()`  
- No API sync in V1

### 4.7 Dashboard (V1: Unified View)
- Same for all roles  
- Widgets:
  - Services scheduled (filter: day/week/month/quarter)  
  - Missed services  
  - Completed services awaiting review (can approve or flag)  
  - Consumables required (filter: period)  
  - Low stock alerts  
  - Services ready to invoice  
  - Contracts expiring (filter: period)  
- All widgets shown to all users

## 5. Non-Functional Requirements (V1)

| Requirement | Detail |
|-----------|--------|
| **Availability** | 24/7 access (planned maintenance <2h/month) |
| **Performance** | Dashboard loads in <2s with 10k+ records |

| **Data Integrity** | All stock changes traceable (who, when, what) |
| **Offline Support** | Not required (web-only, real-time sync) |
| **Compatibility** | Chrome, Firefox, Safari, Edge (latest 2 versions) |
| **Mobile View** | Responsive design, usable on tablets |

## 6. V2 Roadmap (Architect For, Do Not Build)

| Feature | Purpose | V1 Data Model Implications |
|--------|--------|----------------------------|
| **Mobile App** | Field teams check-in/out, mark complete, return stock | Add `check_in_time`, `check_out_time`, `client_signature_url`, `signed_by` |
| **Barcode Scanning** | Scan equipment/consumables in/out | Reserve `barcode`, `qr_code` on equipment and consumables |
| **Sage One Sync** | Auto-push invoices to accounting | Add `sage_invoice_id`, `accounting_integration` table, `outbound_webhooks` queue |
| **Geofencing** | Prompt check-in when near client site | Store `latitude`, `longitude` on clients and services |

---

## 7. V3 Roadmap (Future-Proofing Only)

| Feature | Purpose | V1 Data Model Implications |
|--------|--------|----------------------------|
| **Route Planner** | Optimize daily service order | Add `service_priority`, `estimated_duration`, `planned_order` |
| **Role Switcher** | User views app as different role | Add `role_assignment_source`, `role_changes` audit table |
| **Custom Dashboards** | Role-specific views | Isolate permission logic in `permissions.js` |
| **Offline Mode** | Field teams work without connectivity | Design API for sync-on-reconnect |
| **Photo Evidence** | Upload images of installed equipment | Add `photos` table: `service_id`, `image_url`, `caption` |
